# Lab 3 - Event Subscriptions

In this lab, you will create a WxCC event subscription for **`task:ended`** and deliver the event notifications to a **Webex Connect webhook flow**.

When a contact ends, WxCC sends a webhook notification to Webex Connect. Webex Connect parses the event payload, extracts `data.taskId`, and passes that interaction ID into the Search API and Bad Experience detection logic in later labs.

???+ tip "API Docs"
    The WxCC **Subscriptions** API is documented here: <a href="https://developer.webex.com/webex-contact-center/docs/api/v1/subscriptions" target="_blank">Subscriptions API</a>. The webhook behavior is described here: <a href="https://developer.webex.com/webex-contact-center/docs/using-webhooks" target="_blank">Using Webhooks</a>.

???+ warning
    Bruno client is installed on the laptop. Do not use any web versions.

---

???+ purpose "Lab Objectives"
    In Lab 1, you learned how to call WxCC APIs manually from Bruno and directly from a WxCC flow. In Lab 2, you used Webex Connect as an API orchestration layer for an AI Agent fulfillment action.

    In Lab 3, you will configure an **event-driven integration**. Instead of polling WxCC to check whether an interaction has ended, you will create a subscription that sends `task:ended` notifications to Webex Connect.

    By the end of this lab, you will have:

    - Opened the imported Webex Connect flow with an inbound webhook trigger.
    - Copied the public webhook URL generated by Webex Connect.
    - Created a WxCC subscription for the `task:ended` event.
    - Placed and ended a test call.
    - Verified that Webex Connect received the event payload and exposed `data.taskId`.

???+ curious "Why `task:ended`?"
    Use `task:ended` because the Search API and bad-experience logic run after the interaction has ended. The event payload provides the `taskId` needed by the downstream API calls.

## Lab 3.3 - Create the WxCC `task:ended` Subscription

In this section, you will use Bruno to tell WxCC to send `task:ended` events to the Webex Connect webhook URL.

???+ tool "Create the Subscription in Bruno"
    1. Open **Bruno** on the lab laptop.
    2. Use the WxCC API collection you configured in Lab 1.
    3. Open the collection **Vars** tab.
    4. Update the `webexConnectWebhookUrl` variable with the webhook URL you copied from Webex Connect.
    5. Open the API request **WxCC - Event Subscribe**.
    6. Confirm the request body uses the webhook variable:

        ```json
        {
          "name": "task:ended",
          "eventTypes": [
            "task:ended"
          ],
          "destinationUrl": "{{webexConnectWebhookUrl}}",
          "orgId": "{{org_id}}",
          "resourceVersion": "task:1.0.0"
        }
        ```

    7. Click **Send**.
    8. Confirm you receive a successful response and copy the returned subscription `id`.
    9. Optional: paste the returned `id` into the collection `subscriptionId` variable so you can use the get/delete subscription requests later.

    !!! warning "Avoid duplicate subscriptions"
        If you send the create request multiple times, you may create multiple subscriptions pointing to the same webhook. If your instructor asks you to clean up duplicates, use the list/delete subscription requests in Bruno.

???+ webex "Verify the Subscription Exists"
    1. In Bruno, use the GET request **WxCC - Event List Subscriptions**.
    2. Locate and confirm your `task:ended` subscription.

    !!! success "Checkpoint"
        WxCC is now configured to publish `task:ended` events to Webex Connect.

## Lab 3.4 - Trigger a `task:ended` Event

Now you will place a real call into the contact center, end it, and confirm that the subscription fires.

???+ webex "Place and End a Test Call"
    1. Sign in to the **Agent Desktop** using the **User1** credentials provided for your POD.
    2. Set the agent to **Available**.
    3. From your phone, dial the **Entry Point number** provided in your POD credentials.
    4. Answer the call on the Agent Desktop.
    5. Speak for 15-30 seconds.
    6. End the call.
    7. Submit any wrap-up code on the Agent Desktop.
    8. Wait 10-30 seconds for the webhook event to arrive in Webex Connect.

## Lab 3.5 - Verify the Webhook Payload in Webex Connect

The final validation is to confirm that Webex Connect received the event and can read the `taskId`.

???+ webex "Inspect Transaction Logs"
    1. In Webex Connect, open the **Event Subscription** service.
    2. Open the **Process Events with Search and Campaign** script.
    3. Click the **Debug** icon on the right hand side of the screen, this opens the **Transaction Logs**.
    4. Locate the most recent transaction for your webhook script.
    5. Open the transaction and inspect the inbound webhook payload.
    6. Confirm the payload includes:

        | Field | Why It Matters |
        | :--- | :--- |
        | `type` | Should be `task:ended` |
        | `data.taskId` | Interaction ID used by Lab 4 Search API |
        | `data.orgId` | Organization that generated the event |
        | `data.channelType` | Used later to identify voice interactions |

    !!! success "Checkpoint"
        Webex Connect is now receiving WxCC `task:ended` events. The `taskId` in the payload is available for later labs as the interaction ID.

    ???+ failure "Troubleshooting"
        - **No transaction appears in Webex Connect**: Confirm the script is **Live**, the subscription is active, and the `destinationUrl` exactly matches the webhook URL.
        - **401 or 403 when creating the subscription**: Refresh your Bruno token and confirm your Webex Integration includes `cjp:config`, `cjp:config_read`, `cjp:config_write`, and `cjp:user`.
        - **Subscription exists but no events arrive**: Confirm you ended a real contact and completed wrap-up. Some event delivery can take a few seconds.
        - **Debug log cannot find `taskId`**: Confirm you opened the imported script assigned to your POD. If the `subscription_EndTask` webhook does not expose `inboundWebhook.data.taskId`, stop and ask your instructor to verify the imported script.
        - **Duplicate events arrive**: Check for duplicate active subscriptions pointing to the same webhook URL.
